# Tools

This section describes the `tools/` directory. It is used to **reproduce tables offline** and **generate paper figures** from the **saved judge artifacts**.

---

## 1) Main reproduction script

### `reproduce_valid_evaluations.py`

Purpose:
- Re-generate the **summary tables** from each file’s stored judge records.

I/O:
- **Reads**
  - `supplement/04_results/03_processed_evaluations/*/valid_evaluations/`
- **Writes**
  - `supplement/04_results/03_processed_evaluations/*/summary_tables/`

Properties:
- **Deterministic**: same inputs → same tables
- **Offline**: no API calls, no LLM invocation

---

## 2) Figures (plotting scripts)

Notes:
- Scripts under `figures/` are only for **plotting from processed outputs**; they do not change validity/scoring.
- If you only need the tables, figure scripts are optional.

Figures referenced in the paper are generated by:

- `figures/make_figure1_schema_failure_cliff.py`
  - Plot schema hard-break rate (`A_structure = 0`) across trigger types and models

- `figures/make_figure2_implicit_heatmap.py`
  - Heatmap of mean total scores under implicit triggers (prompt variant × generator model)

- `figures/make_figure3_judge_sensitivity.py`
  - Compare baseline vs paraphrased judges on the same outputs (implicit triggers only)

- `figures/make_figure4_dimension_breakdown.py`
  - Aggregate per-rubric dimension (A–E) means by trigger salience and generator model

- `figures/make_figure5_dimension_failure_rate.py`
  - Compute per-dimension failure rates (non-perfect-score fraction), covering explicit & implicit triggers

- `figures/make_figure6_judge_comparison.py`
  - Extended judge comparison: baseline, paraphrased, schema-strict (implicit triggers)

### Execution order & paths

Run `reproduce_valid_evaluations.py` first to ensure summary tables exist, then run the figure scripts:

- Figure scripts **only read**
  - `supplement/04_results/03_processed_evaluations/<judge_version>/summary_tables/`
- Figure scripts **write PDF figures to**
  - `paper/figures/`
